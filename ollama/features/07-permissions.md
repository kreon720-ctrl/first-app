# 역할 · 권한 매트릭스

TEAM WORKS의 모든 기능은 "누가 할 수 있고, 누가 할 수 없는지"가 명확히 나뉘어 있습니다. 이 문서는 권한 규칙을 한눈에 정리합니다.

## 두 가지 역할

| 역할 | 설명 |
|------|------|
| **팀장(LEADER)** | 팀을 만든 사람. 팀당 1명. 팀 생성과 동시에 자동 지정됩니다. |
| **팀원(MEMBER)** | 팀장의 승인으로 합류한 사용자. |

- 한 사용자가 여러 팀에 속할 수 있으며, 팀마다 역할이 다를 수 있습니다.
- 팀장 양도·팀장 교체 기능은 현재 안내되어 있지 않습니다.

## 황금 규칙

> **"팀 구성원 누구나 만들 수 있다. 수정·삭제는 만든 사람 본인만."**
>
> 예외: 공지사항은 작성자 본인 또는 팀장(LEADER)이 삭제할 수 있습니다.

이 한 문장만 기억하면 대부분의 권한 질문에 답할 수 있습니다.

## 기능별 권한 매트릭스

| 기능 | 팀장(LEADER) | 팀원(MEMBER) |

| 채팅 송수신 | O | O |
| 업무보고(WORK_PERFORMANCE) 전송 | O | O |

**굵게 표시된 항목**이 팀장과 팀원의 권한이 달라지는 지점입니다.

## 팀장 전용 기능 (4가지)

1. **가입 신청 승인/거절** — 공개 팀에 들어온 가입 신청을 팀장만 처리합니다.
2. **나의 할 일 조회** — 내가 팀장인 모든 팀의 대기 신청을 한 화면에서 봅니다.
3. **업무보고 열람 권한 관리** — 어떤 팀원이 업무보고를 볼 수 있는지 지정합니다.
4. **타인 공지사항 삭제** — 작성자가 아니어도 팀장이면 공지를 정리할 수 있습니다.

이 네 가지 외의 모든 기능은 팀장과 팀원이 동등하게 쓸 수 있습니다.

## 업무보고 열람 권한 세부 규칙

- **기본 상태(허용 목록 비어 있음)**: 전체 공개 — 모든 팀원이 모든 업무보고를 볼 수 있습니다.
- **특정 팀원만 허용**: 허용된 팀원과 팀장만 업무보고 열람 가능.
- **팀장은 항상 예외적으로 전체 열람** — 허용 목록과 상관없이 볼 수 있습니다.
- 설정 변경은 즉시 반영됩니다(다음 채팅 조회부터).

## 권한 오류 메시지

권한 없는 동작을 시도하면 아래 메시지가 나타납니다.

| 메시지 | 발생 조건 |
|--------|----------|
| "팀장만 가입 신청을 처리할 수 있습니다" | 팀원이 승인/거절 시도 |
| "일정 수정 권한이 없습니다" (403) | 타인 일정 수정/삭제 시도 |
| "프로젝트 삭제 권한이 없습니다" | 타인 프로젝트 수정/삭제 시도 |
| "포스트잇 수정 권한이 없습니다" | 타인 포스트잇 수정/삭제 시도 |
| "팀장만 권한을 설정할 수 있습니다" | 팀원이 업무보고 열람 권한 설정 시도 |
| "권한이 없습니다" (403) | 공지사항 삭제 시 작성자·팀장 아닌 경우 |

## 자주 묻는 사례

- **Q**: 팀장이니까 팀원이 만든 일정도 당연히 지울 수 있는 거 아닌가요?
  **A**: 아닙니다. 일정·프로젝트·포스트잇은 만든 사람 본인만 수정·삭제할 수 있어요. 팀장도 예외가 아닙니다. 유일한 예외는 공지사항 삭제뿐이에요.

- **Q**: 내가 팀장인데 다른 팀에서는 팀원이에요. 이게 가능한가요?
  **A**: 네, 맞아요. 역할은 팀마다 독립적이라 한 계정이 팀 A에서는 팀장, 팀 B에서는 팀원일 수 있습니다.

- **Q**: 팀원끼리 서로 업무보고를 볼 수 있게 하려면요?
  **A**: 팀장이 `[업무보고 열람 권한 관리]`에서 허용 팀원을 지정해야 합니다. 모두에게 공개하려면 허용 목록을 빈 상태로 저장하면 돼요.
