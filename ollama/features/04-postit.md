# 포스트잇 (날짜별 메모)

## 개념

포스트잇은 캘린더의 특정 날짜에 붙이는 **컬러 메모**입니다. "내일 거래처 미팅 전에 이거 꼭 챙기기!" 같은 짧은 할 일이나 체크리스트를 날짜에 붙여두면 캘린더를 볼 때 함께 표시되어 바로 눈에 들어옵니다.

## 색상

총 5가지 색상을 고를 수 있습니다.

| 색상 코드 | 활용 예시 |
|-----------|----------|
| indigo (기본) | 일반 메모 |
| blue | 참고 사항 |
| emerald | 완료·확인 |
| amber | 준비 항목 |
| rose | 긴급·중요 |

색상 의미는 강제 규정이 아니라 팀에서 자유롭게 정해 쓰시면 됩니다.

## 포스트잇 만들기 (팀원 누구나 가능)

1. 캘린더에서 포스트잇을 붙일 **날짜를 선택**합니다.
2. `[포스트잇 추가]` 버튼을 누릅니다.
3. 생성 폼에서 아래를 입력합니다.
   - **색상**: 5가지 중 선택 (기본값 amber)
   - **내용**: 메모 텍스트 (빈 값 불가)
4. `[저장]` 버튼을 누르면 해당 날짜에 포스트잇이 등록됩니다.

### 권한
- **팀장(LEADER)·팀원(MEMBER) 모두** 포스트잇을 만들 수 있습니다.
- 같은 날짜에 여러 개를 붙일 수 있습니다.

## 포스트잇 보기

- 날짜를 선택해 포스트잇 목록을 조회하면 그 날짜에 등록된 모든 팀 포스트잇을 볼 수 있습니다.
- 작성자, 색상, 내용이 표시됩니다.

## 포스트잇 수정 (작성자 본인만)

자신이 만든 포스트잇의 색상이나 내용을 언제든 바꿀 수 있습니다.

1. 포스트잇을 클릭해 상세를 엽니다.
2. 편집 모드로 전환해 색상·내용을 수정합니다.
3. 저장합니다.

### 권한 경계
- **작성자 본인**만 수정할 수 있습니다.
- 다른 사람이 만든 포스트잇에는 `[수정]` 버튼이 표시되지 않습니다.

## 포스트잇 삭제 (작성자 본인만)

본인이 만든 포스트잇만 떼어낼 수 있습니다. 다른 사람이 만든 포스트잇은 팀장이라도 지울 수 없습니다. 수정·삭제 버튼이 본인 것에만 보입니다.

## 주의사항

- **내용이 비어 있으면 저장할 수 없습니다.** 클라이언트 유효성 검증에서 막힙니다.
- 포스트잇은 특정 날짜에 귀속되므로, 날짜 자체를 바꾸려면 기존 포스트잇을 지우고 다른 날짜에 새로 만드시면 됩니다.
- 알림 기능은 지원하지 않습니다.

## 자주 묻는 사례

- **Q**: 포스트잇은 팀 전체에게 보이나요?
  **A**: 네. 특정 날짜에 누가 붙였든 팀원 전체가 볼 수 있습니다. 작성자 이름도 함께 표시됩니다.

- **Q**: 비공개 메모로 쓸 수 있나요?
  **A**: 포스트잇은 팀 공용입니다. 개인 전용 메모 기능은 현재 안내되어 있지 않습니다.

- **Q**: 색상 의미가 팀 내에서 헷갈려요.
  **A**: 팀 내부에서 "빨강(rose)은 긴급, 파랑(blue)은 참고" 같은 약속을 공지사항에 올려두면 좋습니다.
