# 프로젝트 · 간트차트 · 세부 일정

## 개념: 3단계 계층

TEAM WORKS의 프로젝트 관리는 **세 단계 계층**으로 구성됩니다. 큰 흐름에서 디테일로 점점 좁혀가는 구조입니다.

| 단위 | 역할 | 예시 |
|------|------|------|
| 프로젝트 | 업무의 큰 틀 | "2분기 마케팅 캠페인" |
| 프로젝트 일정 | 간트차트 한 행에 해당하는 업무 덩어리 | "SNS 콘텐츠 기획" |
| 세부 일정 | 업무를 더 작게 쪼갠 할 일 | "인스타 썸네일 디자인" |

## 공통 규칙

- **누구나 생성 가능**: 팀 구성원이라면 팀장·팀원 구분 없이 프로젝트·프로젝트 일정·세부 일정을 만들 수 있습니다.
- **수정·삭제는 생성자 본인만**: 팀장이라도 다른 사람이 만든 건 고치거나 지울 수 없습니다.
- **연쇄 삭제**: 상위를 삭제하면 하위가 모두 함께 삭제됩니다. 프로젝트를 삭제하면 안에 있는 프로젝트 일정과 세부 일정이 전부 사라지고, 프로젝트 일정을 삭제하면 그 아래 세부 일정이 사라집니다.
- **진행률**: 0~100% 사이 숫자로 직접 입력. 자동 계산이 아닙니다.
- **지연 표시**: 오늘이 종료일을 지났는데 진행률이 100%가 아니면 자동으로 '지연' 상태가 되어 색상으로 구분됩니다.

## 프로젝트 생성

1. 팀 메인 화면이나 프로젝트 목록(`/teams/[팀ID]/projects`)으로 이동합니다.
2. `[프로젝트 생성]` 버튼을 누릅니다.
3. 아래 값을 입력합니다.
   - **프로젝트명**(필수, 최대 200자)
   - **설명**(선택)
   - **시작일**·**종료일**(종료일 ≥ 시작일)
   - **담당자**(텍스트, 선택)
4. `[저장]`을 누릅니다. 초기 진행률은 0%, 단계(phase) 목록은 빈 상태로 생성됩니다.

## 프로젝트 단계(phase) 설정

프로젝트 안의 큰 흐름을 "기획 → 개발 → 출시"처럼 단계로 나눌 수 있습니다.

1. 프로젝트 상세에서 `[단계 추가]` 또는 설정 메뉴로 들어갑니다.
2. 단계 이름을 입력하고 순서를 정합니다.
3. 저장하면 각 프로젝트 일정을 이 단계에 연결할 수 있게 됩니다.

단계를 꼭 설정할 필요는 없으며, 단계 없이도 프로젝트 일정을 만들 수 있습니다.

## 프로젝트 수정·삭제

- **수정**: 프로젝트 생성자가 `[수정]`으로 이름·설명·기간·담당자·진행률·단계 구성을 바꿀 수 있습니다.
- **삭제**: 생성자가 `[삭제]`를 누르면 확인 후 프로젝트와 하위의 모든 일정이 함께 삭제됩니다. 삭제는 되돌릴 수 없으니 주의하세요.
- 생성자가 아닌 사람은 수정·삭제 버튼이 보이지 않으며 API 직접 호출 시 "프로젝트 삭제 권한이 없습니다" 같은 오류가 납니다.

## 프로젝트 일정 (간트차트 한 행) 생성

프로젝트 상세 안에서 실제 업무 단위를 추가합니다.

1. 프로젝트 간트차트 화면에서 `[일정 추가]` 버튼을 누릅니다.
2. 입력값
   - **제목**(필수, 최대 200자)
   - **시작일·종료일**(종료일 ≥ 시작일)
   - **담당자**·**색상**(기본 indigo)·**단계(phase)** 선택(설정한 경우)
3. 저장하면 간트차트에 행(row)으로 표시됩니다. 기간 막대의 길이가 기간에 비례합니다.

### 수정·삭제
- 생성자만 가능. 제목·기간·담당자·색상·단계·진행률·지연 여부를 변경할 수 있습니다.
- 삭제 시 그 하위 세부 일정이 함께 사라집니다.

## 세부 일정 생성

프로젝트 일정을 더 작게 쪼갤 때 사용합니다.

1. 프로젝트 일정 옆의 `[세부 일정 추가]` 버튼을 누릅니다.
2. 제목·기간·담당자·색상을 입력해 저장합니다.
3. 해당 프로젝트 일정 하위에 표시되며, 진행률이나 지연 여부도 각각 관리할 수 있습니다.

## 진행률과 지연 관리

- 각 항목을 만든 사람이 진행률을 **0, 10, 25, 50, 75, 100 등 숫자로 직접 갱신**합니다.
- 상위(프로젝트·프로젝트 일정)의 진행률도 직접 입력하는 값입니다. 하위를 기준으로 자동 합산하지 않습니다.
- **지연 강조**: 오늘 날짜가 종료일을 넘겼는데 아직 100%가 아닌 항목은 시각적으로 눈에 띄게 표시되어 "이거 빨리 처리해야겠다"를 바로 알아챌 수 있습니다.

## 자주 묻는 사례

- **Q**: 제가 만든 프로젝트인데 팀장이 못 고친대요. 왜 그래요?
  **A**: 맞습니다. 프로젝트·프로젝트 일정·세부 일정은 모두 "생성자 본인만" 수정·삭제할 수 있어요. 팀장 권한도 예외가 아니에요.

- **Q**: 진행률이 자동으로 계산되나요?
  **A**: 아니요. 각 항목의 진행률은 담당자가 직접 입력하셔야 해요.

- **Q**: 프로젝트를 잘못 지웠는데 복구할 수 있나요?
  **A**: 복구 기능은 안내되어 있지 않아요. 삭제는 신중히 결정하시고, 하위 일정이 함께 사라진다는 점 꼭 기억해 주세요.

- **Q**: 단계(phase) 없이 프로젝트 일정만 추가해도 되나요?
  **A**: 네, 단계는 선택사항이에요. 단계 없이도 프로젝트 일정을 만들 수 있고, 나중에 단계를 추가해 연결할 수도 있어요.

- **Q**: 담당자는 반드시 팀원이어야 하나요?
  **A**: 담당자 필드는 자유 텍스트 형식이에요. 이름을 적는 용도로 주로 쓰이며, 시스템 계정과 연결되지는 않아요.
