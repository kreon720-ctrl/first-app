# TEAM WORKS 사용법 압축 레퍼런스

Gemma 2 9B 컨텍스트 창(8K)에 전체를 주입해도 여유가 남도록 압축한 단일 문서입니다. RAG 없이 매 턴 프롬프트에 붙여 써도 됩니다.

---

## 1. 앱 정체

TEAM WORKS는 팀 단위 캘린더·일정·프로젝트·채팅을 한 화면에서 관리하는 올인원 협업 앱입니다. 슬랙·노션·구글캘린더·엑셀을 오가지 않고 한 곳에서 팀 빌딩부터 업무 완주까지 처리하도록 설계되었습니다.

## 2. 역할과 황금 규칙

### 역할
- **팀장(LEADER)**: 팀을 만든 사람. 팀당 1명. 팀 생성과 동시에 자동 지정.
- **팀원(MEMBER)**: 팀장의 승인으로 합류한 사용자.
- 한 사용자는 여러 팀에 속할 수 있습니다. 팀마다 역할이 다를 수 있습니다.

### 황금 규칙
- **생성**은 팀 구성원(LEADER·MEMBER) 누구나 가능.
- **수정·삭제**는 만든 사람 본인만 가능.
- **예외**: 공지사항만 "작성자 본인 또는 팀장(LEADER)" 삭제 가능.

## 3. 회원가입·로그인

1. 이메일·비밀번호로 회원가입.
2. 이메일 형식이 맞지 않거나 이름이 50자 초과면 오류.
3. 로그인 후 JWT 기반 세션이 발급되며 모든 기능 접근 가능.
4. 미로그인 상태에서 앱 기능 사용 시도 시 로그인 화면으로 이동.

## 4. 팀 관리

### 팀 생성
1. 팀 목록 홈(`/`)에서 `[+ 팀 생성]` 클릭.
2. 팀 생성 화면(`/teams/new`)에서 팀 이름(1~100자) 입력.
3. `[생성]` 누르면 팀이 만들어지고 **본인이 자동 LEADER**가 됨.
4. 팀을 공개(isPublic)로 설정하면 탐색 목록에 노출됨.

### 공개 팀 가입 신청
- **TEAM WORKS에는 초대 기능이 없음.**
- 팀 목록 홈에서 `[팀 탐색]` → `/teams/explore` → 원하는 팀의 `[가입 신청]` 클릭.
- 이미 구성원이거나 PENDING 신청이 있으면 중복 신청 불가(409).

### 가입 신청 승인/거절 (팀장 전용)
- 팀장은 `[나의 할 일]`(`/me/tasks`)에서 여러 팀의 대기 중 신청을 한 번에 확인.
- `[승인]`: 신청자가 즉시 팀원(MEMBER)으로 등록됨.
- `[거절]`: 팀 합류 없이 처리 종료. 신청자에게 별도 알림 없음.
- 팀원이 승인/거절 시도 시 "팀장만 가입 신청을 처리할 수 있습니다" 오류.

### 팀 탈퇴·양도·삭제
- 현재 안내되어 있지 않음.

## 5. 캘린더 · 팀 일정

### 화면 구성
- 팀 메인 `/teams/[팀ID]`: 좌측 캘린더 + 우측 채팅.
- 데스크탑(≥1024px)은 분할 화면, 모바일은 `[캘린더]`/`[채팅]` 탭 전환.

### 뷰
- 월간(기본) / 주간 / 일간 탭, `<` `>`로 전·후 이동.
- 날짜 셀 클릭 시 오른쪽 채팅 영역도 그 날짜로 이동.

### 일정 CRUD
- **생성**: `[+ 일정 추가]` → 제목(필수, 최대 200자) / 설명 / 시작 일시 / 종료 일시(> 시작). 팀원 누구나 가능.
- **수정·삭제**: 생성자 본인만. 다른 사람 일정에는 `[수정]`·`[삭제]` 버튼이 나타나지 않음. 팀장 예외 없음.
- 종료 ≤ 시작이면 "종료 시각은 시작 시각 이후여야 합니다" 오류.

### 알림
- 일정 변경 시 푸시/이메일 알림은 현재 안내되어 있지 않음.

## 6. 포스트잇 (날짜별 컬러 메모)

- 캘린더 날짜에 붙이는 짧은 컬러 메모.
- 5색: indigo(기본 일반) / blue(참고) / emerald(완료) / amber(준비, 생성 시 기본값) / rose(긴급).
- **생성**: 날짜 선택 → `[포스트잇 추가]` → 색상 선택 + 내용(빈 값 불가).
- **수정·삭제**: 작성자 본인만. 팀장도 타인 포스트잇 불가.
- 같은 날짜에 여러 개 붙일 수 있음. 팀 공용이라 작성자 이름과 함께 표시됨.

## 7. 프로젝트 (간트차트)

### 3단계 계층
| 단위 | 설명 |
|------|------|
| 프로젝트 | 업무의 큰 틀. 예: "2분기 마케팅 캠페인" |
| 프로젝트 일정 | 간트차트 한 행. 예: "SNS 콘텐츠 기획" |
| 세부 일정 | 업무를 더 쪼갠 할 일. 예: "인스타 썸네일 디자인" |

### 공통 규칙
- 팀원 누구나 생성, 수정·삭제는 생성자 본인만.
- **연쇄 삭제**: 프로젝트 삭제 → 하위 프로젝트 일정 + 세부 일정 모두 삭제.
- **진행률**: 0~100 숫자 **수동 입력**. 자동 합산 아님.
- **지연 표시**: 오늘이 종료일을 지났는데 100% 미만이면 자동 '지연' 강조.

### 프로젝트 생성
1. 팀 메인 또는 `/teams/[팀ID]/projects`에서 `[프로젝트 생성]`.
2. 입력: 프로젝트명(필수, 최대 200자), 설명, 시작일·종료일(종료 ≥ 시작), 담당자(자유 텍스트).
3. 저장 시 초기 진행률 0%, 단계(phase) 빈 목록.

### 단계(phase)
- "기획 → 개발 → 출시" 같은 큰 흐름을 직접 정의. 선택 사항.
- 프로젝트 일정 생성 시 단계에 연결 가능.

### 프로젝트 일정 생성
1. 프로젝트 상세 화면에서 `[일정 추가]`.
2. 입력: 제목(필수), 시작일·종료일, 담당자, 색상(기본 indigo), 단계 선택.
3. 간트차트에 행으로 표시됨.

### 세부 일정 생성
- 프로젝트 일정 옆의 `[세부 일정 추가]` → 제목·기간·담당자·색상 입력.

## 8. 채팅 (3종류)

### 타입
| 타입 | 누가 보내나 | 누가 보나 |
|------|-------------|----------|
| 일반 대화 (NORMAL) | 팀 구성원 누구나 | 팀 전체 |
| 업무보고 (WORK_PERFORMANCE) | 팀 구성원 누구나 | 기본 팀장만. 팀장이 허용한 팀원 추가 |
| 공지사항 (Notice) | 팀 구성원 누구나 | 팀 전체, 채팅 상단 고정 |

### 공통 동작
- 메시지는 KST 기준 작성 날짜별로 자동 그룹핑.
- 캘린더에서 날짜 클릭 시 채팅도 그 날짜로 이동.
- **갱신은 3~5초 폴링 방식**. 본인 메시지는 즉시 표시, 다른 팀원 메시지는 최대 5초 지연.
- 한 메시지 최대 2,000자. 빈 메시지 전송 불가.
- **메시지 수정·개별 삭제 기능은 안내되어 있지 않음.**
- **파일/이미지 첨부는 안내되어 있지 않음.**

### 일반 대화
- 입력창에 쓰고 `[전송]` 또는 Enter. 특별한 분리 없이 팀 전원에게 표시.

### 업무보고 전송
1. `[업무보고]` 버튼으로 모드 전환.
2. 내용 입력 후 `[전송]`.
3. 일반 대화와 시각적으로 구분되어 표시.

### 업무보고 열람 권한
- 팀장은 항상 모든 업무보고 열람 가능.
- 팀원은 기본적으로 타인의 업무보고 열람 불가.
- 팀장이 `[업무보고 열람 권한 관리]`에서 허용 팀원 지정 시 그 팀원도 열람 가능.
- **허용 목록을 빈 상태로 저장하면 전체 공개**(모든 팀원 열람 가능).
- 팀원이 권한 설정 시도 시 "팀장만 권한을 설정할 수 있습니다" 오류.

### 공지사항
- `[공지 작성]` → 내용(최대 2,000자) → `[등록]`. 채팅 상단에 고정.
- 팀 구성원 누구나 작성 가능.
- **삭제**: 작성자 본인 **또는 팀장(LEADER)** 가능 — 다른 기능과 다른 유일한 예외.
- 이유: 오래되거나 부적절한 공지를 팀장이 정리할 수 있도록.

## 9. 화면 경로 요약

| 화면 | 경로 | 접근 |
|------|------|------|
| 팀 목록 홈 | `/` | 로그인 사용자 |
| 팀 생성 | `/teams/new` | 로그인 사용자 |
| 공개 팀 탐색 | `/teams/explore` | 로그인 사용자 |
| 나의 할 일 | `/me/tasks` | 팀장만 의미 있음 |
| 팀 메인(캘린더+채팅) | `/teams/[팀ID]` | 해당 팀 구성원 |
| 프로젝트 목록 | `/teams/[팀ID]/projects` | 해당 팀 구성원 |
| 일정 생성 | `/teams/[팀ID]/schedules/new` | 해당 팀 구성원 |

## 10. 반응형 동작

- 데스크탑(≥1024px): 캘린더 + 채팅 분할 화면.
- 모바일(<640px): 상단 탭으로 `[캘린더]`/`[채팅]` 전환.
- 화면 크기를 바꿔도 선택된 날짜는 유지.

## 11. 자주 혼동되는 포인트

- **팀 초대 기능 없음** — 공개 팀 탐색 + 가입 신청만 지원.
- **푸시·이메일 알림 없음** — 사용자가 앱을 직접 확인.
- **메시지 수정/삭제 없음** — 정정은 새 메시지로. 공지사항만 삭제 가능.
- **파일 첨부 없음** — 텍스트만 지원.
- **진행률 자동 계산 안 됨** — 직접 입력.
- **팀장도 타인 창작물 수정 불가** — 유일한 예외는 공지사항 삭제.
- **팀 삭제·탈퇴·양도 없음**.

## 12. 오류 메시지 해석

| 메시지 | 원인 | 해결 |
|--------|------|------|
| "이미 해당 팀의 구성원입니다" | 이미 가입한 팀에 재신청 | 가입 신청 불가 |
| "이미 가입 신청이 진행 중입니다" | PENDING 신청 존재 | 팀장 승인·거절 대기 |
| "이미 처리된 가입 신청입니다" | 승인/거절 완료된 건 재처리 | 새 신청이 필요한 경우 신청자가 다시 신청 |
| "팀장만 가입 신청을 처리할 수 있습니다" | 팀원이 승인/거절 시도 | 팀장에게 요청 |
| "일정 수정 권한이 없습니다" (403) | 타인 일정 수정 시도 | 생성자에게 요청 |
| "프로젝트 삭제 권한이 없습니다" | 타인 프로젝트 삭제 시도 | 생성자만 가능 |
| "권한이 없습니다" (공지 삭제 시 403) | 작성자도 팀장도 아닌 사람의 삭제 시도 | 작성자나 팀장에게 요청 |
| "종료 시각은 시작 시각 이후여야 합니다" | 시간 순서 오류 | 종료 > 시작으로 수정 |
| "제목은 필수입니다" | 제목 빈 값 | 제목 입력 |

## 13. 비기능 참고

- 서버 타임존: KST(UTC+9).
- 비밀번호 저장: bcrypt 해싱.
- 인증: JWT Access + Refresh Token.
- 채팅 응답 목표: 500ms 이하.
- 동시 접속: 팀당 최소 50명 지원.
