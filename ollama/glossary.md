# TEAM WORKS 용어 사전

챗봇이 사용자의 구어체 표현과 내부 코드/문서상의 용어를 매끄럽게 연결할 수 있도록 정리한 한/영 대조 사전입니다.

## 역할·구성원

| 한글 표현 | 내부 용어 | 설명 |
|-----------|-----------|------|
| 팀장 | LEADER | 팀을 만든 사람, 팀당 1명, 자동 지정 |
| 팀원 | MEMBER | 팀장의 승인으로 합류한 사용자 |
| 사용자 | User | 로그인 계정을 가진 사람 |
| 팀 구성원 | TeamMember | 팀에 속한 사용자(LEADER/MEMBER 포함) |

## 팀 관리

| 한글 표현 | 내부 용어 | 설명 |
|-----------|-----------|------|
| 팀 | Team | 협업 단위 |
| 팀 이름 | name | 1~100자 |
| 팀 설명 | description | 최대 500자, 선택 |
| 공개 팀 | isPublic = true | 탐색 목록에 노출됨 |
| 비공개 팀 | isPublic = false | 탐색 목록에 비노출 |
| 가입 신청 | TeamJoinRequest | 공개 팀에 들어가려고 보내는 요청 |
| 대기 중 | PENDING | 팀장이 아직 승인/거절 안 한 상태 |
| 승인됨 | APPROVED | 팀장이 승인, 팀원으로 등록됨 |
| 거절됨 | REJECTED | 팀장이 거절 |
| 나의 할 일 | My Tasks | 팀장이 받은 가입 신청을 모아 보는 화면(`/me/tasks`) |

## 일정

| 한글 표현 | 내부 용어 | 설명 |
|-----------|-----------|------|
| 일정 | Schedule | 팀 캘린더에 올리는 일정 |
| 시작 일시 | startAt | 일정 시작 시각 |
| 종료 일시 | endAt | 일정 종료 시각(startAt보다 뒤) |
| 제목 | title | 최대 200자 |
| 설명 | description | 선택 |
| 월간 뷰 | month view | 한 달 단위 |
| 주간 뷰 | week view | 한 주 단위 |
| 일간 뷰 | day view | 하루 단위 |

## 포스트잇

| 한글 표현 | 내부 용어 | 설명 |
|-----------|-----------|------|
| 포스트잇 | Postit | 날짜에 붙이는 컬러 메모 |
| 색상 | color | indigo / blue / emerald / amber(기본) / rose |
| 내용 | content | 빈 값 불가 |
| 날짜 | date | 포스트잇이 붙은 날짜 |

## 프로젝트

| 한글 표현 | 내부 용어 | 설명 |
|-----------|-----------|------|
| 프로젝트 | Project | 업무의 큰 틀 |
| 프로젝트 일정 | ProjectSchedule | 간트차트 한 행 |
| 세부 일정 | SubSchedule | 프로젝트 일정의 하위 할 일 |
| 단계 | phase | "기획 → 개발 → 출시" 같은 프로젝트 내부 흐름 |
| 진행률 | progress | 0~100 숫자, 수동 입력 |
| 지연 | isDelayed | 종료일 지남 + 진행률 100% 미만이면 true |
| 담당자 | manager / leader | 자유 텍스트 |
| 시작일 | startDate | Date 형식 |
| 종료일 | endDate | Date 형식, 시작일 이상 |

## 채팅

| 한글 표현 | 내부 용어 | 설명 |
|-----------|-----------|------|
| 채팅 메시지 | ChatMessage | 팀 채팅에 올라오는 메시지 |
| 일반 대화 | NORMAL | 기본 채팅 타입, 팀 전체 노출 |
| 업무보고 | WORK_PERFORMANCE | 팀장 기본 열람, 민감 보고용 |
| 공지사항 | Notice | 상단 고정, 작성자·팀장 삭제 가능 |
| 업무보고 열람 권한 | WorkPerformancePermission | 팀장이 팀원에게 부여하는 열람 권한 |
| 폴링 | polling | 3~5초 간격 자동 갱신 |
| 실시간 | real-time | 폴링 방식이므로 완전 실시간은 아님 |

## 화면 경로

| 한글 표현 | 경로 |
|-----------|------|
| 팀 목록 홈 | `/` |
| 팀 생성 | `/teams/new` |
| 공개 팀 탐색 | `/teams/explore` |
| 나의 할 일 | `/me/tasks` |
| 팀 메인(캘린더+채팅) | `/teams/[팀ID]` |
| 프로젝트 목록 | `/teams/[팀ID]/projects` |
| 일정 생성 | `/teams/[팀ID]/schedules/new` |

## 권한 키워드

| 한글 표현 | 의미 |
|-----------|------|
| 생성자 본인만 | 만든 사람(createdBy)만 수정/삭제 가능 |
| 팀장 전용 | LEADER만 수행 가능 |
| 팀원 누구나 | LEADER/MEMBER 모두 가능 |
| 황금 규칙 | "생성은 누구나, 수정·삭제는 만든 사람만" (공지사항만 예외) |

## 오류 코드

| 코드 | 의미 |
|------|------|
| 400 | 입력값 오류(제목 빈 값, 기간 역전 등) |
| 401 | 로그인 필요 |
| 403 | 권한 없음(타인 것 수정/삭제 등) |
| 404 | 대상 없음 |
| 409 | 중복(이미 가입됨, 이미 PENDING 신청 존재 등) |

## 타임존·인증

| 한글 표현 | 내부 용어 | 설명 |
|-----------|-----------|------|
| 한국 표준시 | KST (UTC+9) | 서버 기준 타임존. 채팅 날짜 그룹핑 기준 |
| JWT | JSON Web Token | 인증 토큰 방식 |
| Access Token | 단기 인증 토큰 | 요청 시 동봉 |
| Refresh Token | 장기 갱신 토큰 | Access Token 재발급용 |
| bcrypt | 비밀번호 해시 알고리즘 | 평문 비밀번호 저장 안 함 |
