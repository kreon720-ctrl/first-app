# TEAM WORKS 챗봇 학습/참조 문서 모음

TEAM WORKS 앱 사용법을 안내하는 챗봇을 로컬 Gemma 2 9B(Ollama) 기반으로 구동하기 위한 문서 세트입니다.

## Gemma 2 9B 최적화 포인트

- **컨텍스트 창 8K 토큰**: 한 번에 주입할 수 있는 문서량이 제한적이라 문서를 "덩어리 단위"로 쪼개 놓았습니다.
- **한국어 품질**: 모델이 한국어를 이해하지만 영어 대비 토큰 효율이 낮으므로 문서를 간결하게 구성했습니다.
- **지시 준수력**: `temperature 0.3`, 예시 기반 톤 고정으로 일관된 답변을 유도합니다.
- **RAG 친화적**: 각 `features/*.md`는 독립적으로 읽어도 완결되도록 작성되어 문서 검색(RAG)에 바로 쓰기 좋습니다.

## 3가지 활용 시나리오

### ① 가장 간단한 구성 — Modelfile만 이용
```bash
cd ollama
ollama create teamworks-gemma -f Modelfile
ollama run teamworks-gemma
```
`Modelfile`에 `SYSTEM` 블록으로 핵심 사용법이 이미 주입되어 있어 별도의 검색 없이 바로 답변할 수 있습니다. 컨텍스트가 가장 가볍게 쓰이는 방식입니다.

### ② RAG 구성 — features/* + glossary 를 임베딩
- 각 `features/*.md`와 `glossary.md`를 청크로 사용합니다.
- 사용자 질문으로 유사도 검색 → 상위 1~3개 청크만 프롬프트에 주입합니다.
- 시스템 프롬프트는 `Modelfile`의 `SYSTEM` 블록을 재사용하거나 `system-prompt.md` 내용을 활용합니다.

### ③ 풀 컨텍스트 구성 — knowledge-base.md 주입
- 8K 토큰 한도 내에서 `knowledge-base.md` 전체(약 2,800 토큰)를 사용자 메시지와 함께 매 턴 주입합니다.
- 검색 인프라 없이도 앱 전반에 대한 질문에 답할 수 있습니다.
- 장점: 구성 단순. 단점: 매 턴 토큰 소비.

## 답변 품질 유지 팁

1. **사용자 질문을 그대로 모델에 넘기지 마세요.** 시스템 프롬프트의 규칙(예: "3~5문장", "버튼은 `[...]`로")이 무시되지 않도록 매 호출에 시스템 프롬프트를 붙입니다.
2. **답이 문서에 없으면 추측하지 않게 하세요.** `system-prompt.md`의 거부 패턴("안내되어 있지 않아요")을 지키도록 few-shot 예시를 함께 주입합니다.
3. **일관된 한국어 존댓말을 유지하세요.** 예시 답변들이 모두 존댓말이라 모델도 따라갑니다.
4. **최신 정보가 필요하면 문서를 갱신하세요.** 코드 변경 시 해당 `features/*.md`만 고치면 됩니다.
